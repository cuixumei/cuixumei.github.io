GitHub+Hexo搭建个人博客

Hexo是一个快速、简洁且高效的博客框架，Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。GitHub Pages是由 GitHub 官方提供的一种免费的静态站点托管服务,Hexo解析渲染成html页面后，通过GitHub 仓库托管和发布自己的静态网站页面完成博文的发布。下面正式从零开始搭建属于自己的第一个网站。

准备工作

Node+Git+注册github账号

创建仓库

1.创建GitHub Pages 仓库

1.Repository name 中输入 你自己的GitHub用户名.github.io
2.勾选 “Initialize this repository with a README”

2.设置用户名和邮箱

3.创建及添加SSH秘钥

然后一直回车，结束后添加秘钥。
进入 [C:\Users\用户名.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。
在 GitHub 中进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。复制 id_rsa.pub 的内容到 Key 中，点击 Add SSH key 完成添加。

4.验证连接

创建项目

1.安装hexo-cli

2.初始化项目

3.安装依赖

4.启动项目

至此，博客框架已生成，目录结构如下：

_config.yml：网站的 配置 信息，您可以在此配置大部分的参数。
package.json：应用程序的信息。
scaffolds：模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。
source：资源文件夹是存放用户资源的地方。
themes：主题 文件夹。

新建博客

您可以在命令中指定文章的布局（layout），默认为 post，可以通过修改 _config.yml 中的 default_layout 参数来指定默认布局。
接下来就可以按照自己想法填充博客内容了，基本的操作参考官方文档。
